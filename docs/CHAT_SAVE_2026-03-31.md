# Chat Save - 2026-03-31

## Scope
Saved summary of the session-auth and telnet input integration guidance discussed in chat.

## Core Architecture Decisions

- `MudSession` and `MudDbAccount` should be separate entities.
- A session references account identity after auth via `session->account_id`.
- Do not merge session/account structs.

## Input Pipeline (Client -> Session)

1. `read_cb` in `src/net/mud_listener.c` receives socket bytes.
2. `mud_telnet_recv(conn, data, len)` in `src/net/mud_telnet.c` parses telnet stream.
3. `TELNET_EV_DATA` calls `mud_input_recv(conn, ...)`.
4. `mud_input_recv` assembles line text and calls `mud_input_on_line(conn, line)`.
5. `mud_input_on_line` should call `mud_session_on_line(conn->session, line)`.
6. `mud_session_on_line` dispatches by `session->state`.

## Important Rule

- No `scanf` for auth/session logic.
- Password/name input must come from telnet line callbacks (`line` argument passed to `mud_session_on_line`).

## `mud_session_on_line` Responsibilities

- Validate `session`, `session->conn`, `line`, and ownership (`conn->session == session`).
- Update activity tracking each line:
  - `session->last_active_at = time(NULL)`
  - `mud_connection_touch(session->conn)`
- Dispatch by `session->state`.
- Call helper handlers for each state.
- Trigger `mud_session_transition(...)` for state changes.

## `mud_session_transition` Responsibilities

- Be the only place that mutates `session->state`.
- Perform enter-state side effects only (prompts, telnet echo mode, disconnect close path).
- Avoid blocking calls in transition flow (`sleep` etc.).
- Avoid DB auth logic here; keep DB auth in line handlers.

## `session_lookup_account` Guidance

Recommended shape:

- Inputs: `MudSession*`, input username (or use `session->username`), output `MudDbAccount*`.
- Validate pointers and DB open state.
- Normalize/validate username.
- Query with `mud_db_account_get_by_name(...)`.
- On found, sync session fields:
  - `session->account_id`
  - `session->failed_logins`
- Return explicit result (`FOUND`, `NOT_FOUND`, `INVALID_INPUT`, `DB_ERROR`) if possible.

## `session_authenticate_password` Guidance

- Input: `MudSession* session`, `const char* input_password`.
- Lookup account by `session->username`.
- Verify with `mud_crypto_verify_password(input_password, account.password_hash)`.
- On success:
  - set `session->account_id = account.id`
  - set `session->failed_logins = 0`
  - update login timestamp/counters via DB helpers
  - wipe sensitive temp buffers (`mud_crypto_memzero`)
- On failure:
  - increment `session->failed_logins`
  - increment DB failed counter
  - return `false`

## Compile Blockers Previously Identified

- `mud_net_loop_run()` treated as bool in `main.c` though it returns `void`.
- Invalid manual session allocation block in `mud_listener.c` (`sizzeof`, type misuse).
- Multiple structural and syntax errors in `mud_session.c` transition/auth code.
- Invalid field/member usage (e.g., `session->player->username` where `player` is `void*`, `session->id` missing).
